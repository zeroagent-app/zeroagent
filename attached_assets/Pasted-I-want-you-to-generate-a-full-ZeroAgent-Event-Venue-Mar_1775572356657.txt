# 1. Project Overview

ZeroAgent is a **digital marketplace for booking event venues** (weddings, functions, large gatherings) with these key features:

- Users can **browse verified venues** with details: images, videos, pricing, capacity, location, and availability calendar.
- Users can **send booking requests** for a selected date.
- Owners **approve or reject booking requests**, after which an in-app chat unlocks.
- Admins can **monitor all users, venues, and bookings**, approve/remove listings, and track payment status manually.
- Role-based access: **Admin / Owner / User (Tenant)**.

- **Database**: Supabase (Postgres)

# 4. Functional Requirements

## Users

- Signup / Login  
- Roles: **Admin, Owner, User**  
- Users can browse venues and send booking requests.

## Owners

- Add/Edit/Delete venue listings  
- Upload images/videos  
- Set price, capacity, location  
- Manage incoming booking requests  
- Only approved users can chat (voice notes optional placeholder)

## Bookings

- Users send a **booking request** for a specific date  
- Owners approve/reject requests  
- Chat unlocks only after approval

## Admin

- View all users, bookings, venues  
- Approve or remove listings  
- Track payment status (paid/unpaid) manually
